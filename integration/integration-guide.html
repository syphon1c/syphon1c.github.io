AI Security Gateway - Enterprise Integration Guide â€‹

This document provides comprehensive guidance for configuring enterprise integrations with SIEM, Slack, and email systems for real-time security alert forwarding and incident management.

Integration Overview â€‹

The AI Security Gateway supports multiple enterprise integrations to forward security alerts and notifications to external systems:

  • SIEM Integration: Forward security events to Splunk, Elastic, QRadar, and other SIEM platforms
  • Slack Notifications: Real-time alert notifications to Slack channels
  • Email Alerts: Send alert summaries via email (SMTP)
  • Langfuse: AI observability and tracing integration

All integrations are configured through the Web Admin Interface at Settings â†’ Integrations.

The Integrations tab provides a unified interface for configuring all external system integrations

Accessing Integration Settings â€‹

  1. Navigate to the AI Security Gateway Web UI (default: http://localhost:8080)
  2. Log in with your admin credentials
  3. Click Settings in the left navigation menu
  4. Select the Integrations tab

From this interface, you can:

  • Enable/disable each integration
  • Configure connection settings
  • Test connectivity
  • View integration status and health

Configuration Overview â€‹

All integration settings are stored securely in the database and can be managed through the Web UI. Sensitive credentials (API keys, passwords) are encrypted at rest.

SIEM Integration â€‹

Supported SIEM Platforms â€‹

The AI Security Gateway can forward security events to any SIEM platform that supports HTTP-based event ingestion:

  • Splunk: Via HTTP Event Collector (HEC)
  • Elastic Stack: Via Elasticsearch REST API
  • IBM QRadar: Via REST API endpoints
  • ArcSight: Via HTTP receiver
  • Generic SIEM: Any platform supporting JSON over HTTP

Configuring SIEM Integration â€‹

Via Web UI (Recommended):

  1. Navigate to Settings â†’ Integrations
  2. Locate the SIEM Integration section
  3. Enable the integration toggle
  4. Configure the following settings:
    • Endpoint: SIEM API endpoint URL (e.g., https://splunk.company.com:8088/services/collector)
    • API Key: Authentication token/key from your SIEM platform
    • Index: Target index or destination for events
    • TLS Verify: Enable/disable SSL certificate verification
  5. Click Test Connection to verify settings
  6. Click Save Settings to apply changes

SIEM Setup Examples â€‹

Splunk HEC:

  1. In Splunk: Settings â†’ Data Inputs â†’ HTTP Event Collector
  2. Create new token with desired index (e.g., ai_security)
  3. Copy the token and endpoint URL
  4. Enter in AI Security Gateway SIEM settings

Elastic Stack:

  1. Create API key: POST /_security/api_key
  2. Create index template for event storage
  3. Use endpoint format: https://elasticsearch:9200/ai-security/_doc

IBM QRadar:

  1. Generate SEC token: Admin â†’ Authorized Services
  2. Create custom event property for AI Security Gateway
  3. Configure log source for HTTP events

Slack Integration â€‹

Setup Overview â€‹

Slack integration provides real-time notifications for security alerts directly to your team's Slack channels.

Creating a Slack Webhook â€‹

  1. Create Slack App:

    • Go to https://api.slack.com/apps
    • Click "Create New App" â†’ "From scratch"
    • Name: "AI Security Gateway"
    • Select your workspace

  2. Enable Incoming Webhooks:

    • Navigate to "Incoming Webhooks"
    • Turn on "Activate Incoming Webhooks"
    • Click "Add New Webhook to Workspace"
    • Select channel (e.g., #security-alerts)
    • Copy the webhook URL

  3. Configure in AI Security Gateway:

    • Navigate to Settings â†’ Integrations
    • Locate the Slack Integration section
    • Enable the integration toggle
    • Paste the webhook URL
    • Configure channel name and display settings
    • Set minimum severity level for notifications
    • Click Test Connection to verify
    • Click Save Settings

Configure the minimum severity level for Slack notifications:

  • LOW: All alerts (not recommended for production)

Email Integration â€‹

Configuring Email Alerts â€‹

Via Web UI:

  1. Navigate to Settings â†’ Integrations
  2. Locate the Email Integration section
  3. Enable the integration toggle
  4. Configure SMTP settings:
    • SMTP Host: Your mail server hostname
    • SMTP Port: Usually 587 (TLS) or 465 (SSL)
    • SMTP Username: Email account username
    • SMTP Password: Email account password or app password
    • Use TLS: Enable for secure connections
    • From Address: Sender email address
    • Recipients: Comma-separated list of recipient emails
    • Minimum Severity: Alert severity threshold
    • Format: HTML or plain text
  5. Click Test Connection to verify settings
  6. Click Save Settings

Email Provider Setup Examples â€‹

Gmail:

  1. Enable 2-Step Verification: https://myaccount.google.com/security
  2. Generate App Password:
    • Go to https://myaccount.google.com/apppasswords
    • Select "Mail" and "Other (Custom name)"
    • Name: "AI Security Gateway"
    • Copy the 16-character password
  3. Use in AI Security Gateway:
    • SMTP Host: smtp.gmail.com
    • SMTP Port: 587
    • Use TLS: Enabled
    • Username: Your Gmail address
    • Password: The 16-character app password

Microsoft 365 / Outlook:

  • SMTP Host: smtp.office365.com
  • SMTP Port: 587
  • Use TLS: Enabled
  • Username: Your Microsoft 365 email
  • Password: Your account password

SendGrid (API-based):

  • SMTP Host: smtp.sendgrid.net
  • SMTP Port: 587
  • Use TLS: Enabled
  • Username: apikey
  • Password: Your SendGrid API key

Email Alert Format â€‹

Emails include:

  • Subject: [AI Security Alert] ${Severity}: ${Title}
  • HTML Body (if format: html):
    • Color-coded severity banner
    • Alert details table
    • Link to Web UI
    • Proxy information
    • Policy and rule details
    • Raw data excerpt
  • Plain Text Body (if format: text):
    • Severity, title, description
    • Proxy name, policy, rule ID
    • Timestamp and alert ID

Langfuse Integration â€‹

Langfuse provides AI observability and tracing capabilities for monitoring LLM proxy interactions.

Via Web UI:

  1. Navigate to Settings â†’ Integrations
  2. Locate the Langfuse Integration section
  3. Enable the integration toggle
  4. Configure connection settings:
    • Public Key: Your Langfuse public key
    • Secret Key: Your Langfuse secret key
    • Host: Langfuse instance URL (default: https://cloud.langfuse.com)
  5. Click Test Connection to verify settings
  6. Click Save Settings

For detailed Langfuse setup instructions, see the Langfuse Setup Guide.

Testing Integrations â€‹

Web UI Testing (Recommended) â€‹

  1. Navigate to Settings â†’ Integrations
  2. Configure your integration settings
  3. Click the Test Connection button for the specific integration
  4. Review the test results displayed in the UI
  5. Check for success or error messages

The test will:

  • Verify connectivity to the external system
  • Validate authentication credentials
  • Send a test event/message
  • Report any configuration issues

Generating Test Alerts â€‹

To verify that integrations are working correctly, you can trigger test alerts:

  1. Create a test MCP or LLM proxy
  2. Configure it with a security policy
  3. Send a request that triggers a security alert
  4. Verify the alert appears in your integrated systems (SIEM, Slack, Email)

See the Quick Start Guide for detailed instructions on creating proxies and testing security policies.

Event Types Sent to Integrations â€‹

SIEM Events â€‹

  • Security Alerts: Real-time security alerts from MCP and LLM proxies
  • Proxy Status Changes: Proxy start/stop/restart events
  • Authentication Events: OAuth login/logout, failed authentication attempts
  • Configuration Changes: Policy updates, proxy configuration changes
  • System Events: Backup completion, settings updates, integration status

Slack Notifications â€‹

  • Real-time Security Alerts: Immediate notification for alerts above configured severity threshold
  • Proxy Status Changes: Notifications for proxy lifecycle events
  • Integration Status: Alerts when integrations fail or recover

Email Alerts â€‹

  • Alert Notifications: Individual emails for each alert above configured severity threshold
  • Critical Alerts: Immediate high-priority emails for critical issues

Troubleshooting â€‹

Common Issues â€‹

Authentication Failures:

  • Verify API keys and credentials are correct
  • Check that endpoint URLs are accessible
  • Ensure service accounts have proper permissions
  • Test credentials directly with the external service

Network Connectivity:

  • Verify the endpoint URL is reachable from your server
  • Check firewall rules and network policies
  • Ensure SSL/TLS certificates are valid
  • Test connectivity using curl or similar tools

Integration Not Receiving Events:

  • Verify the integration is enabled in Settings â†’ Integrations
  • Check that the minimum severity threshold is appropriate
  • Review recent test results for errors
  • Ensure proxies are configured with security policies that generate alerts

Viewing Integration Logs â€‹

Integration activity and errors are logged in the system audit logs:

  1. Navigate to Settings â†’ System Logs
  2. Filter by event type: "integration"
  3. Review recent integration attempts and errors
  4. Check timestamps and error messages for troubleshooting

Getting Help â€‹

If you continue to experience issues:

  • Check the integration status indicators in Settings â†’ Integrations
  • Review the system logs for detailed error messages
  • Verify your external service configuration (SIEM, Slack, email server)
  • Test the integration using the built-in test functionality

Best Practices â€‹

  1. Start with Test Connections: Always test integrations before relying on them in production
  2. Set Appropriate Severity Thresholds: Avoid alert fatigue by filtering to relevant severities
  3. Monitor Integration Health: Regularly check the Integrations page for status updates
  4. Secure Credentials: Never commit API keys or passwords to version control
  5. Review Logs Periodically: Check integration logs to ensure events are being sent correctly
  6. Use Dedicated Service Accounts: Create specific accounts for the AI Security Gateway with minimal required permissions
