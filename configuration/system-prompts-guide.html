System Prompts Overview ‚Äã

System Prompts allow administrators to inject additional security controls, instructions, and constraints into LLM requests. These prompts are automatically added to LLM API requests to control model behavior, enforce compliance, and add security guidelines.

Key Features ‚Äã

  • Automatic Injection: System prompts are automatically injected into LLM requests based on proxy or user group assignments
  • Priority-Based Selection: Only the highest priority prompt is injected per request (user group prompts take precedence over proxy prompts)
  • **Template Variables**: Use dynamic variables like `{{.User}}`, `{{.Date}}`, and `{{.Organization}}` in prompt content
  • Per-Proxy Assignment: Assign different prompts to different LLM proxies
  • Per-User Group Assignment: Assign prompts to specific user groups/API keys for fine-grained control
  • Audit Logging: All prompt operations are logged for compliance and security auditing

System Prompts Use Cases ‚Äã

Prompt Security Controls ‚Äã

  • Enforce data protection policies (e.g., "Never reveal API keys or credentials")
  • Prevent information leakage (e.g., "Do not include debug information in responses")
  • Add compliance requirements (e.g., "Always comply with GDPR data protection requirements")

Behavioral Guidelines ‚Äã

  • Professional communication standards (e.g., "Always respond in a professional and courteous manner")
  • Content restrictions (e.g., "Do not generate content that violates company policies")
  • Response formatting (e.g., "Always format code blocks with syntax highlighting")

Organization-Specific Instructions ‚Äã

  • Brand voice and tone (e.g., "Maintain a friendly but professional tone consistent with our brand")
  • Domain-specific knowledge (e.g., "When discussing medical topics, always include appropriate disclaimers")
  • Custom workflows (e.g., "Always include a summary section at the end of your response")

Priority System ‚Äã

How Priority Works ‚Äã

Important: Only one system prompt is injected per LLM request. The system selects the prompt based on the following priority order:

  1. User Group Prompts (highest priority) - If a user group has a prompt assigned, it takes precedence
  2. Proxy Prompts (fallback) - If no user group prompt exists, the proxy-level prompt is used

Assignment Behavior ‚Äã

Single Assignment Per Target: Only one system prompt can be assigned to a proxy or user group at a time. When you assign a new prompt:

  • Existing assignments are automatically replaced - The old prompt assignment is removed and replaced with the new one
  • No manual removal required - You don't need to remove the old assignment before assigning a new one
  • Same prompt updates priority - If you assign the same prompt again, only the priority is updated

This behavior ensures that only one prompt is active per proxy or user group, matching the injection behavior where only one prompt is used per request.

Priority Resolution ‚Äã

Since only one prompt can be assigned per proxy or user group, priority resolution primarily applies when determining which prompt to use between user group and proxy levels:

  1. User Group Prompts (highest priority) - If a user group has a prompt assigned, it takes precedence
  2. Proxy Prompts (fallback) - If no user group prompt exists, the proxy-level prompt is used

The assignment priority (0-100) and prompt base priority (0-100) are used when multiple prompts exist at the same level, but in practice, only one prompt is assigned per target, so these priorities are mainly used for ordering in the UI and for potential future multi-prompt support.

Creating System Prompts ‚Äã

System Prompts Via Web Interface ‚Äã

  1. Navigate to Security ‚Üí Security Prompts in the sidebar
  2. Click Create Prompt
  3. Fill in the form:
    • Name: A descriptive name for the prompt (e.g., "Professional Response Policy")
    • Description: Brief description of what the prompt does
    • Content: The actual prompt text to inject
    • Priority: Set priority using the slider (0-100, Low/Medium/High)
    • Active: Toggle to enable/disable the prompt
  4. Click Create

System Prompts ‚Äã

Template Variables ‚Äã

System prompts support dynamic template variables that are automatically substituted at request time.

Available Variables ‚Äã

  • {{.User}} - The username of the authenticated user (if OAuth is enabled)
  • {{.Date}} - Current date in RFC3339 format
  • {{.Organization}} - Organization name (if configured)
  • {{.ProxyName}} - Name of the proxy handling the request
  • {{.ProxyID}} - ID of the proxy handling the request

Example with Template Variables ‚Äã

You are an AI assistant for {{.Organization}}.
Today is {{.Date}}.
The user {{.User}} is requesting assistance.
Please maintain professional communication standards.

Rendered Output:

You are an AI assistant for Acme Corporation.
Today is 2025-01-04T14:30:00Z.
The user john.doe@acme.com is requesting assistance.
Please maintain professional communication standards.

Assigning Prompts ‚Äã

Assign to Proxy ‚Äã

Important: Only one system prompt can be assigned to a proxy at a time. Assigning a new prompt will automatically replace any existing assignment.

Via Web Interface:

  1. Navigate to Proxies ‚Üí Select a proxy ‚Üí Edit
  2. Scroll to System Prompts section
  3. Select a prompt from the dropdown
  4. Set priority (optional, defaults to 0)
  5. Click Assign Prompt

Note: If a prompt is already assigned to this proxy, assigning a new one will replace it.

Assign to User Group ‚Äã

Important: Only one system prompt can be assigned to a user group at a time. Assigning a new prompt will automatically replace any existing assignment.

Via Web Interface:

  1. Navigate to Teams & API Keys ‚Üí Select a user group
  2. Click System Prompts tab
  3. Click Assign Prompt
  4. Select a prompt from the dropdown
  5. Set priority using the slider (0-100)
  6. Click Assign Prompt

Note: The modal includes a note explaining that only the highest priority prompt will be injected. User group prompts take precedence over proxy prompts. If a prompt is already assigned to this user group, assigning a new one will replace it.

Edit Prompt ‚Äã

Via Web Interface:

  1. Navigate to Security ‚Üí Security Prompts
  2. Click Edit on the prompt you want to modify
  3. Update fields as needed
  4. Click Update

Activate/Deactivate ‚Äã

You can toggle prompts on/off without deleting them:

  • Active: Prompt is eligible for injection
  • Inactive: Prompt is not injected (but assignments are preserved)

Delete Prompt ‚Äã

Warning: Deleting a prompt also removes all assignments. This action cannot be undone.

System Prompts Best Practices ‚Äã

1. Use Clear, Specific Instructions ‚Äã

  • ‚úÖ Good: "Never include API keys, passwords, or credentials in your responses."
  • ‚ùå Bad: "Be secure."

2. Set Appropriate Priorities ‚Äã

  • Use higher priorities (70-100) for critical security policies
  • Use medium priorities (40-69) for behavioral guidelines
  • Use lower priorities (0-39) for general instructions

3. Test Before Deployment ‚Äã

  • Create prompts with low priority first
  • Test with sample requests
  • Monitor logs to ensure prompts are injected correctly

4. Use Template Variables Wisely ‚Äã

  • Template variables are substituted at request time
  • Ensure variables are available (e.g., `{{.User}}` requires OAuth)
  • Test with different user contexts

5. Document Your Prompts ‚Äã

  • Use descriptive names and detailed descriptions
  • Document why a prompt exists and what it enforces
  • Keep prompts updated as requirements change

6. Monitor Assignment Counts ‚Äã

  • Check the "Assignments" column to see where prompts are used
  • Remember that assigning a new prompt automatically replaces the old one
  • Review assignments regularly for security compliance

7. Replacing Existing Assignments ‚Äã

  • When you assign a new prompt to a proxy or user group, the old assignment is automatically removed
  • You don't need to manually remove the old assignment first
  • This ensures only one prompt is active per proxy or user group at any time

System Prompts Security Considerations ‚Äã

Content Validation ‚Äã

  • Prompts are validated for size (max 10,000 characters)
  • Potentially malicious patterns are logged (but not blocked)
  • Review audit logs regularly for suspicious prompt content

Prompt Access Control ‚Äã

  • Only administrators can create, edit, and delete prompts
  • Only administrators can assign prompts to proxies or user groups
  • All operations are logged in the audit system

Priority Conflicts ‚Äã

  • User group prompts always override proxy prompts
  • Only one prompt can be assigned per proxy or user group (new assignments automatically replace old ones)
  • Review priority settings to avoid unintended overrides

System Prompts Troubleshooting ‚Äã

Prompt Not Being Injected ‚Äã

  1. Check Prompt Status: Ensure the prompt is marked as "Active"
  2. Check Assignment: Verify the prompt is assigned to the proxy or user group
  3. Check User Group: If using API keys, verify the user group has a prompt assigned (user group prompts override proxy prompts)
  4. Check Logs: Review audit logs for prompt injection events

Replacing an Existing Assignment ‚Äã

Note: Only one prompt can be assigned to a proxy or user group at a time. When you assign a new prompt:

  • The old assignment is automatically removed
  • You don't need to manually remove it first
  • Simply assign the new prompt and it will replace the existing one

Multiple Prompts Not Working ‚Äã

Note: Only one prompt is injected per request. If you need multiple instructions, combine them into a single prompt:

You are a helpful AI assistant. Follow these guidelines:
1. Always respond professionally
2. Never reveal sensitive information
3. Include appropriate disclaimers when needed

Template Variables Not Substituting ‚Äã

  1. Check Variable Name: Ensure correct syntax {{.VariableName}}
  2. Check Availability: Some variables (like {{.User}}) require OAuth to be enabled
3. **Check Logs**: Review logs for template substitution errors
